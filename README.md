## Часть 1

Пайплайн разделён на пять независимых этапов, что обеспечивает изоляцию сбоев, параллельное выполнение и удобство отладки:
- Подготовка среды и кеширование – инициализация раннера, восстановление артефаков предыдущих сборок.
- Конфигурация – генерация файлов сборки с учётом целевого компилятора.
- Компиляция – трансляция исходного кода в объектные файлы и линковка исполняемого модуля.
- Тестирование – верификация корректности возвращаемых кодов и формата вывода.
- Упаковка и публикация – сохранение бинарных артефактов для последующей доставки или ручной проверки.

Часть 2. Безопасное управление секретами

Для обеспечения безопасной передачи конфиденциальных данных в целевое приложение реализована интеграция с HashiCorp Vault. Процесс взаимодействия выстроен по принципу Zero-Trust:
- Сервис-скрипт (demo_vault.sh) аутентифицируется в локальном экземпляре Vault.
- Запрашивается секрет по пути secret/data/ci/cpp-lab (KV Secrets Engine v2).
- Значение динамически экспортируется в переменную окружения API_KEY.
- C++-приложение извлекает данные через std::getenv(), не выводя их в стандартный поток вывода.
- Переменная принудительно очищается из памяти оболочки после завершения выполнения.

Тут я запускал vault локально у себя на устройстве, запускал программу и она подтягивала оттуда по ключу значение, пытался через nginx и добавление секретов в actions на github, но что-то задушился и решил оставить как есть
